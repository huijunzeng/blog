一个实体类在多个系统中唯一，在客户端创建VO接收来自服务端的对象

优点：
      如果更新的字段影响当前进程的业务，只需要更新客户端的VO及唯一的服务端实体类，如果更新的字段不影响当前进程的业务，则只需更新唯一的服务端实体类。且不影响分库

缺点：
      增加进程之间通讯的频率，增加团队之间协作的成本
